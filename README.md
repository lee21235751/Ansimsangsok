# 안심상속 Vercel 배포 준비본 v0.9.4

이 프로젝트는 v0.7.4 운영 배포 직전 기준을 Vercel/GitHub 배포용으로 정리한 정적 사이트입니다.

## 포함 범위

- 랜딩페이지
- 무료 진단
- Supabase 리드 저장
- 샘플 리포트 PDF
- 결제 오픈 대기 신청
- 로컬 관리자 화면(`#admin`)
- 사업자 리스크/수익모델 체크 문서

## 아직 포함하지 않는 범위

- 실제 카드결제/PG
- 운영용 인증 관리자 조회
- 전문가 실제 추천/배정
- 법률자문/세무자문/유언장 작성 대행

## 로컬 실행

```powershell
cd C:\Projects\Ansimsangsok-Deploy
npm.cmd run build
$env:PORT=5203
npm.cmd start
```

브라우저:

```text
http://localhost:5203/
```

## Vercel 환경변수

Vercel Project Settings → Environment Variables에 아래 값을 등록합니다.

```text
VITE_SUPABASE_URL
VITE_SUPABASE_ANON_KEY
VITE_SUPABASE_LEADS_TABLE
VITE_ADMIN_ACCESS_KEY
```

`VITE_SUPABASE_ANON_KEY`에는 Supabase의 publishable key 또는 legacy anon public key를 넣습니다. service_role/secret key는 넣지 않습니다.

## 배포 원칙

공개 사이트는 Supabase에 리드 insert만 수행합니다. 운영용 리드 전체 조회는 공개 프론트엔드에 열지 않고, 추후 인증된 서버 API 또는 Supabase Edge Function으로 분리합니다.

## v0.9.4 고객 화면 완전 정리

- PWA/앱 설치 기능은 추가하지 않았습니다.
- 모바일 웹 가독성 방향은 유지했습니다.
- 고객 화면에서 내부 운영 문구를 제거했습니다.
- “사업자 리스크”, “검증”, “결제 기능 붙이기 전”처럼 고객 신뢰를 떨어뜨릴 수 있는 표현을 삭제했습니다.
- “결제 전 상품 선택”을 “관심 있는 리포트를 선택해주세요”로 변경했습니다.
- “결제 오픈 대기” 표현을 “리포트 준비 알림”으로 정리했습니다.
- 정책/제공 범위 문구는 고객용으로 압축하고, 자세한 내용은 접힘 영역으로 정리했습니다.
- 법률자문·세무자문·유언장 작성 대행·전문가 추천이 아니라는 고지는 유지했습니다.
